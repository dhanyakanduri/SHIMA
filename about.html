About Secure Hashed Identity Message Authentication

Secure Hashed Identity Message Authentication (SHIMA) is a web-based authentication system designed to enhance the security of online platforms by leveraging modern cryptographic techniques. The system securely processes and verifies user credentials using hashing algorithms to ensure that sensitive data, such as passwords, are never stored in plaintext and cannot be easily compromised.

Project Overview

This project aims to implement a robust, secure authentication mechanism where users can log in using their username and password. These credentials are hashed and securely transmitted to the server for validation. The use of hashing algorithms ensures that even if the system is compromised, the stored passwords remain protected.

Key Features

  • Username and Password Authentication: Users provide their username and password to authenticate securely.
  • Hashing for Security: Passwords are hashed using secure algorithms like SHA-256 before being transmitted to the server.
  • Message Authentication: Ensures data integrity during communication by hashing the data payload.
  • Data Privacy: No sensitive information, like raw passwords, is stored in the database.

How It Works

When a user logs into the system, their password is immediately hashed on the client side before it is sent to the server. The server then compares this hashed password with the hashed version stored in the database. If the hashes match, the user is authenticated successfully.

In addition to hashed passwords, we also incorporate Message Authentication Codes (MAC) to validate that the login data has not been tampered with in transit. This provides an additional layer of security against man-in-the-middle attacks.

Technologies Used

  • HTML5 & CSS3: For creating a responsive and user-friendly interface.
  • JavaScript: For client-side hashing and form validation.
  • PHP: For server-side authentication and redirection.
  • OpenSSL: For cryptographic functions such as hashing.
  • SQL: For secure data storage of user credentials.

Future Enhancements

  • Multi-factor authentication for improved security.
  • Integration with OAuth 2.0 for third-party authentication providers.
  • Implementation of session-based token systems for improved session management.
